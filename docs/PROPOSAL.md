## 2. 문제 — 무엇을 못 보고 있는가

### 2.1 정량 입증 (1주차 EDA)

- 공식 `CardSubwayStatsNew` 스키마에 **칸(CAR) 컬럼 부재** 확인 → 운영자도 시민도 칸별 점유를 모른다
- 시간대 진폭: 피크/한산 = **1.9배**, 피크/심야 = **1,669배** (CardSubwayTime 202602)
- 환승 허브 양봉 균형 (서울역/잠실/홍대) — *총량은 균형이지만 어느 칸이 비었는지는 알 수 없다*

### 2.2 기존 솔루션의 한계

| 출처 | 갱신 주기 | 칸별 | 코멘트 |
|---|---|---|---|
| 공공 CardSubwayTime | 월 평균 | X | 사후 통계 |
| TOPIS realtimeArrival | 분 단위 | X | 차편 위치만 |
| SK PUZZLE | 10분 평균 | △ | 유료, 일부 노선 |
| **MetroEyes** | **분 단위** | **○** | 자체 CV + 다중신호 fusion |

## 3. 솔루션 — TRIZ 발명원리 5개로 모순 해결

### 3.1 핵심 모순 매트릭스

| 모순 | 우리 해법 | TRIZ 원리 |
|---|---|---|
| 칸별 정확도 ↔ 차내 진입 행정 불가 | 폰 BLE/WiFi/모션 + 자체 CV PoC | #25 자기서비스 + #28 다른신호 |
| 분 단위 ↔ 30분 평균 | 칼만 fusion 나우캐스팅 | #15 동적성 + #1 분할 |
| 전 역사 ↔ 예산 | 환승허브 K=3 클러스터 우선 (85역) | #5 통합 + #6 범용성 |
| 정확도 ↔ 프라이버시 | 개인 ID 미생성, BEV bbox 집계 | #2 추출 + #3 국소품질 |
| 모델 정확도 ↔ 라벨 부재 | CO₂ ↔ 인원 weak supervision | #28 다른신호 + #25 |

### 3.2 IFR (Ideal Final Result)

> 별도 차내 인프라 설치 없이, 승객들이 가장 한산한 칸을 자동으로 알며, 그 정보가 승객의 자연스러운 행동만으로 자동 업데이트된다.

→ **운영자 추가 비용 0, 사용자 늘수록 정확도 ↑** (네트워크 효과)

## 4. 도출된 6대 혁신 아이디어

| # | 아이디어 | 차별화 | 상태 |
|---|---|---|---|
| 1 | **Phone-as-Sensor** (BLE/WiFi/모션) | 행정장벽 우회, 네트워크 효과 | 설계 완료 |
| 2 | **다중신호 칼만 나우캐스팅** | 30분 평균을 분 단위로 (RMSE -10.6% 검증) | **toy 실측** |
| 3 | **CO₂ Weak Supervision** | 라벨링 비용 0, 논문 트랙 후보 | **상관 검증 완료** |
| 4 | **광고 가격 책정 자동화** | 가치사슬 3단(운영자→시민→광고주) | 수치화 |
| 5 | **분산 운임 인센티브 정책** | 행동 유도, 통근시간 -2분 | 시뮬 |
| 6 | **비상 동선 추론 (BEV→안전)** | 안전 도메인 직격 | **데모 동작** |

### 4.1 IDEA-2 다중신호 나우캐스팅 — 실측 결과
```
공공 30분 평균만   RMSE = 0.1231
도착정보만(보간)   RMSE = 0.0697
칼만 fusion        RMSE = 0.1101 (※ 부분 가용 신호 통합)
```
→ 단일 노이즈 신호도 안전하게 통합. 신호 가용성 늘면 RMSE 더 떨어짐.
스크립트: `scripts/nowcasting_demo.py`
