개발 방법론

1) 가설 → 데이터 검증 → 모델: 각 모듈 수치 검증 후 통합 (ex: CO₂ 가설 → Pearson r 검정 → weak supervision 신호 채택)

2) MVP 우선: 합성 시뮬 → 실 카메라 PoC → Cloudflare 외부 노출. 욕심 부리지 않음.

3) Production-grade 인프라: GitHub Pages 자동 배포 + Cloudflare named tunnel → 발표 당일 라이브 시연 가능.

4. 개발과정 및 방법

핵심 알고리즘 — 호모그래피 BEV 변환

① 4점 캘리브레이션

카메라 영상의 ground plane 4점(시계방향) → BEV 평면 [0,1]² 정규화 좌표로 매핑. cv2.findHomography (RANSAC) 로 3×3 변환 행렬 산출.

② 검출 → 트래킹 → 투영

YOLO11s 사람 검출 → BoT-SORT ID 추적 (Kalman + ReID) → bbox 발 위치 (xc, y_bottom) → 호모그래피로 BEV 변환.

③ 점유 정규화

BEV 좌표 → 칸 정원으로 normalize (0~1.05). 분 단위 칼만 필터로 노이즈 평활.

# 호모그래피 + BEV 투영 (의사코드)
img_pts = [(240,130), (400,130), (560,360), (80,360)]
bev_pts = [(0.30,0.05), (0.70,0.05), (0.95,0.95), (0.05,0.95)]
H = cv2.findHomography(img_pts, bev_pts, RANSAC)
# 매 프레임
tracks = yolo.track(frame, persist=True, tracker='botsort.yaml')
for t in tracks:
  foot = (t.xc, t.y2)
  bev = H @ [foot.x, foot.y, 1]
  ws.send({id, bev_x, bev_y, class})
# 칼만 fusion (TRIZ #15+#1)
x = predict(x, P, Q)
for z, R in [avg30, arrival, cv]:
  K = P / (P + R)
  x = x + K * (z - x)
