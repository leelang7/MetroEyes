## 1. 본질 문제 (Ideal Final Result)

**IFR**: *별도 차내 인프라 설치 없이, 승객들이 가장 한산한 칸을 자동으로 알며, 그 정보가 승객의 자연스러운 행동만으로 자동 업데이트된다.*

→ **폰이 곧 센서, 행동이 곧 라벨, 운영자 추가 비용 0**.

## 3. 도출된 혁신 아이디어 9선

### IDEA-1. **Phone-as-Sensor (TRIZ #25 자기 서비스)**
승객 폰 BLE 핑 / WiFi probe RSSI / 가속도계 진동 → 차내 카메라 0대로 점유 추정.

- **혁신성**: 행정 장벽(차내 진입) 우회. 사용자 늘수록 정확도 ↑ (네트워크 효과)
- **선행연구**: WiFi probe request 기반 군중 추정은 학계에 있지만, *지하철 칸별*은 새로움
- **구현 경로**: Flutter 백그라운드 — `flutter_blue_plus` + `sensors_plus` → BevSocket 텔레메트리 채널 추가
- **데모 가능성**: D-10 內 4명이서 강의실에서 PoC 가능

### IDEA-2. **공공 30분 평균 → 분 단위 나우캐스팅 (TRIZ #15 동적성 + #1 분할)**
SK PUZZLE 10분 + CardSubway 30분 + realtimeArrival 분 단위 + 자체 CV 1초 단위 → 칼만 필터로 통합 추정.

- **혁신성**: 공공 데이터의 갱신 주기 한계를 *수학적으로* 해결. "30분 평균을 1분 단위로 본다"
- **수치 근거**: 평일 18시 환승 허브 양봉 변동성 EDA → 분 단위 진폭 큼 (시간대 진폭 1.9배)
- **데모**: `scripts/nowcasting_demo.py` 에서 toy 칼만 필터로 시연

### IDEA-3. **CO₂ Weak Supervision (TRIZ #28 + #25)**
지하역사 CO₂ IoT 센서 (서울 환경부 무료 데이터) ↔ 승하차 인원 상관 → BEV 모델 자기지도 학습.

- **혁신성**: 라벨링 비용 0. 사람 많으면 CO₂ ↑ — 물리법칙이 곧 라벨
- **검증 필요**: 1주차 미해결 항목, P1 — D-7 내 1일 분량으로 회귀 R² 검증
- **논문 트랙**: 한국멀티미디어학회 weak supervision 트랙 후보

### IDEA-4. **광고 가격 책정 자동화 (가치사슬 한 단 더)**
역세권 시간대 인구 (citydata_ppltn 분 단위) → 옥외광고 시간차 가격 자동 매핑.

- **혁신성**: 운영자(서울교통공사) → 시민 → **광고주** 3단 가치사슬
- **시장 크기**: 한국 옥외광고 시장 1.7조 / 지하철 광고 ~2,000억 / 시간차 가격 +5% = **연 100억 추가 매출**
- **구현**: `frontend/operator_web/ad_pricing.html` 신규 — 역×시간대 매트릭스 + 동적 가격 곡선

### IDEA-5. **분산 운임 인센티브 정책 제안 (TRIZ #15 동적)**
환승 허브 양봉 균형 → 08:30~08:45 환승 시 -100원 즉시 차감. citydata 분 단위 인구로 동적 발동.

- **혁신성**: 단순 정보 제공 → *행동 유도*. ROI 수치화 가능
- **시뮬**: `scripts/policy_roi.py` 에 분산 인센티브 시나리오 — 평균 통근시간 -2분 가정 → 연 1.26조 사회적 가치

### IDEA-6. **비상 동선 추론 (BEV → 안전)**
점유 분포 + 출입문 위치 → 화재/테러 시 자동 대피 경로 제안. CV가 이미 BEV 평면을 갖고 있어 무료.

- **혁신성**: 안전 도메인 직격. 심사위원에게 정책 임팩트 ★★★
- **구현**: `src/cv/tesla_bev.py` 에 `compute_evacuation_path(bev_density, exits)` 추가 — A* 알고리즘
- **데모**: 운영자 콘솔에 "비상 시뮬" 버튼 추가하면 30초 시연

### IDEA-7. **임산부석 점유 양보 알림 (M7)**
칸 양 끝 분홍 좌석 ROI에 시간×인원밀도 통계 → 30초+ 점유 + 동행 미감지 시 양보 권유 송출. 얼굴/번호판 식별 X.

- **혁신성**: 약자 배려 + 프라이버시 양립. 차내 디스플레이가 *비강제* 안내 송출
- **구현**: `lite_server.py` `priority_seat` incident type (cycle 226)

### IDEA-8. **에스컬레이터·통로 병목 사전 경고 (M7)**
좁은 ROI 영역 평균 속도 < 0.3m/s가 45초+ → 우회 안내. 이태원 참사처럼 *압사 전* 단계에서 자동 분산.

- **혁신성**: 안전 사고를 *예측*하는 정책 직접 매핑. 단순 사후 통계 X
- **구현**: `lite_server.py` `bottleneck` incident type (cycle 226)

### IDEA-9. **도착 알림 5중 모달리티 (M8 — 약자·노이즈 캔슬링 사용자 배려)**
시민 폰이 GPS로 도착지 근접을 자기 추출 → 시각 banner + 진동 + 음성 + beep + 시스템 알림을 *동시* 발사.

- **혁신성**: 차내 방송 의존성 제거. 노이즈 캔슬링 / 이어폰 / 유튜브 / 청각 약자 / 폰 락/백그라운드 모두 인지
- **구현**: `passenger_app/index.html` `setDestination` + `fireArrivalAlert` + `_notifyArrival` + `_requestWakeLock` (cycle 286, 291)
- **5중 모달리티 (이중 안전망)**:
  1. 시각 banner flash (scale + shadow)
  2. Web Vibration API 햅틱 — 이어폰 사용자도 진동 인지
  3. Web Audio API beep — sine 합성 (외부 파일 X, 800/1200Hz)
  4. SpeechSynthesis 4언어 (ko-KR/en-US/zh-CN/ja-JP)
  5. **System Notification API** — 앱 백그라운드 / 폰 락 / 다른 앱 사용 중에도 잠금 화면 알림 (arriving은 requireInteraction=true)
- **Wake Lock API**: 도착지 설정 동안 화면 슬립 방지 (visibilitychange 복귀 재획득)
- **3단 임계값**: > 1.5km (안전 거리) / ≤ 1.5km (곧 도착, 60초 cooldown) / ≤ 600m (도착, 30초 cooldown)
- **사회적 가치**: 청각 약자 (한국 등록 청각장애인 42만 명) + 노이즈 캔슬링 사용자 (1,200만 잠재 사용자) → 접근성 제도와 직접 연결 (장애인차별금지법 / 교통약자법)
